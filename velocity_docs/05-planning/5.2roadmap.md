### Hard Constraints

- **Local execution only:** No Azure deployment, no Docker, no CI/CD pipeline. The MVP runs from source code on a local machine using `dotnet run` + `npm run dev` exclusively (W-002, W-003).
- **SQLite only:** SQLite is the only permitted database for the MVP. No cloud or external database server may be introduced (NFR-COMPAT-001).
- **Application-managed authentication:** Registration and login use application-managed local accounts with BCrypt-hashed passwords. Microsoft Entra ID and all corporate SSO providers are excluded from this MVP (W-001).
- **Session-derived identity:** The API must always derive the current user identity from the authenticated server-side session or token mechanism (cookie or JWT — implementation's choice per SAD-DD-001 §5.4 / Assumption A-01). The frontend must never supply a trusted employee or approver identifier for any business decision.
- **Passwords hashed always:** Passwords must never be stored in plain text. BCrypt hashing is mandatory for all accounts, including seeded accounts.
- **Database file not publicly exposed:** The SQLite `.db` file must not be committed to source control and must not contain real credentials. Excluded via `.gitignore` (NFR-SEC-004).
- **Four entities, five states only:** Scope is strictly limited to Employee, Absence Type, Request, and Approval entities, and the five request states (Draft, Submitted, Approved, Rejected, Cancelled). No additional entities may be introduced without a formal scope change.
- **No prohibited screens:** No HR administration screens, no role management UI, and no account administration screens are permitted in this MVP (W-006, W-012).
- **API-enforced business rules:** All business rules — date validation, state transitions, ownership checks, manager authorization — must be enforced by the API, not only by the UI (NFR-SEC-003).
- **No prohibited architectural patterns:** MediatR, CQRS, generic repositories, and messaging frameworks are excluded. Implementation follows the vertical-slice model defined in SAD-DD-001 Key Design Decisions (NFR-MAINT-002).
- **Go-Live fixed at 2026-07-22:** Implementation, hardening, and acceptance sign-off must all be completed on 2026-07-22. No extension is available.
