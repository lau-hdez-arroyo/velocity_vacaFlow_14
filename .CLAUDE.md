
# Project: VacaFlow

VacaFlow (IGS Solutions internal MVP, project ID `VacaFlow_14`) replaces email/Teams/spreadsheet vacation-request handling with a structured, auditable request lifecycle: **Draft → Submitted → Approved / Rejected / Cancelled**, enforced by a Domain-layer state machine. Two roles: **Employee** (manage own requests) and **Manager** (approve/reject requests from assigned employees). No cloud deployment, no orgs/SSO, no notifications, no vacation-balance math in this MVP.

## Planned architecture (from `velocity_docs/07-development/7.1tech-docs.md`, Draft — Awaiting Approval)

Once implementation starts, it must follow this contract exactly (deviations require a documented ADR):

- **Stack:** ASP.NET Core Minimal API on .NET 8 / C# 12, EF Core 8 Code-First (auto-migrate + seed at startup), SQLite (`vacaflow.db`, file-based, gitignored) · Next.js 14 / React 18 / TypeScript frontend, dev proxy `/api/*` → API. No Docker, no external DB server, no cloud services for the MVP.
- **Style: Pragmatic Vertical Slice Onion** (ADR-001) — one file per business operation under `Api/Slices/{Auth,Requests,Approvals}/`, each containing the endpoint, command record, inline `IValidator<T>`, and handler. Application-layer service classes for these operations are prohibited; shared logic goes in `Application/Shared/` or `Domain/`.
- **Onion dependency rule:** `Domain` (entities, `Enums`, `Ports`, `RequestStateMachine` — a static class) has zero dependencies on anything else, no EF Core/ASP.NET Core references. `Infrastructure` depends on `Domain`+`Application`. `Api` and `Web` are the outer entry points.
- **Auth:** ASP.NET Core Cookie Session only (`HttpOnly`, `SameSite=Lax`) — JWT is prohibited for the MVP. Identity must be read from `HttpContext.User` exclusively, never from request body/route params. Passwords hashed via `IPasswordHasher` → `BCryptHasher`, never BCrypt called directly from a slice.
- **Validation:** every slice registers an `IValidator<T>`, invoked by `ValidationPipelineInvoker` before the handler runs — structurally wired, not by convention.
- **State transitions:** every state-changing slice calls `RequestStateMachine.Transition()` before `SaveChangesAsync()`; invalid transitions raise `DomainException` → mapped to HTTP 422 by `GlobalExceptionHandler`. Exactly one `SaveChangesAsync()` per handler (e.g. approval status + `Approval` record commit atomically).
- **Expected commands once bootstrapped:** `cd Api && dotnet run` (port 5000/5001) and `cd Web && npm run dev` (port 3000) in separate terminals; `cd tests && dotnet test` (add `--filter "FullyQualifiedName~Name"` for one test, `Domain.Tests/` etc. to scope a project). Reset local DB by deleting `Api/vacaflow.db` and restarting.

See `Docs/Clean-Architecture-Onion-DotNet-Reference.md` for the general Onion/Clean Architecture audit checklists this project's reduced-Onion pattern is based on.
