## 1. Quality Attributes (from NFR)

Quality attributes identified from the Non-Functional Requirements document (NFR-001 v3.0).

> **NFR Priority scale:** Low = 1–3 · Medium = 4–6 · High = 7–10

| # | Attribute | NFR Priority | Definition |
|---|-----------|--------------|------------|
| 1 | Security | 10 (High) | The system must enforce password hashing, session-based identity derivation, role-based authorization, database file protection, and session management. Plain-text password storage and identity spoofing are hard rejection conditions. |
| 2 | Reliability | 9 (High) | The system must enforce valid state transitions, date validation rules, approval record integrity, and data persistence across restarts. Invalid transitions and corrupt state are rejection conditions. |
| 3 | Availability | 7 (High) | The application must start successfully from source code, complete cold-start migration and seeding without manual intervention, and remain operational throughout the acceptance review session without unplanned crashes. |
| 4 | Maintainability | 7 (High) | The codebase must follow reduced Onion Architecture layer separation, avoid unnecessary abstraction patterns (no MediatR, generic repositories, CQRS frameworks), maintain automated test coverage for critical business logic, and include clear local setup documentation. |
| 5 | Usability | 6 (Medium) | The UI must not display invalid actions for the current request state, must use clear readable labels and human-readable validation messages, and must be navigable by keyboard without critical layout breakage at 150% zoom. |
| 6 | Compatibility | 6 (Medium) | The application must run entirely from source code on a local machine without Docker, cloud services, or external database servers. The frontend must function on the last 2 releases of Chrome, Edge, Firefox, and Safari. |
| 7 | Performance | 5 (Medium) | All user-initiated API calls must respond in under 2 seconds; page navigation must render in under 3 seconds; cold-start migration must complete in under 10 seconds. Measured under single-user local SQLite conditions. |
